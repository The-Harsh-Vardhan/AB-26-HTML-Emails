EMAIL ENGINEERING RULES

Follow professional email development standards.

1. STRUCTURE

• Use a single centered container table (max-width 600px)
• Use nested tables for layout
• Do NOT use div-based layouts
• Do NOT use flexbox or grid
• Avoid margin CSS (use padding instead)
• Avoid position:absolute or complex CSS

2. BUTTONS

Create bulletproof HTML buttons using tables.

Buttons must:
• have background color
• include padding
• include border radius
• include fallback text
• work even if images are blocked

3. IMAGES

• All images must use width="100%" and max-width
• Include alt text
• Avoid very large images
• Ensure hero image scales properly on mobile

4. RESPONSIVE RULES

Use media queries for screens under 600px.

On mobile:
• stack columns vertically
• convert multi-column layouts into single column
• make buttons full width
• reduce padding for smaller screens

5. EMAIL SAFETY

Avoid:
• JavaScript
• web fonts
• external CSS files
• complex selectors

Use only email-safe CSS properties.

6. CLEAN CODE

• Avoid unnecessary wrappers
• Avoid deeply nested tables
• Keep HTML size small to prevent Gmail clipping
• Use consistent spacing and indentation

7. ACCESSIBILITY

• Use readable font sizes
• Include alt text on images
• Maintain sufficient color contrast
